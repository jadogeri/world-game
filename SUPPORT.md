# Support

## Getting Help

- **Documentation** — start with [README.md](README.md) for architecture, setup, and API reference.
- **Contributing** — see [CONTRIBUTING.md](CONTRIBUTING.md) for the development workflow.
- **Bugs** — open an issue describing the problem, steps to reproduce, expected vs. actual behavior, and your environment.
- **Feature requests** — open an issue describing the use case and desired behavior.
- **Security issues** — do **not** open a public issue; follow [SECURITY.md](SECURITY.md) instead.

## Before Asking

1. Check [README.md](README.md) — most setup, configuration, and usage questions are covered there.
2. Check [CHANGELOG.md](CHANGELOG.md) to see if your issue is already known or fixed.
3. Search existing issues for a similar report.

## Response Times

This project is maintained on a best-effort basis. There is no guaranteed response time.
